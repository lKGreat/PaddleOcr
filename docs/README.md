# 文档导航

- 最近更新：2026-02-07
- 维护人：项目维护团队

本目录用于维护项目长期规划、架构演进和产品目标实施闭环，建议每个迭代结束后同步更新。

## 核心文档
- `BUSINESS_DIRECTION.md`：业务方向大纲，说明目标用户、核心场景、阶段规划与业务验收指标。
- `TECHNICAL_ROADMAP.md`：12-18 个月技术路线图，包含阶段里程碑、风险与维护机制。
- `ARCHITECTURE_MAINTENANCE.md`：项目架构维护文档，明确总纲任务、已完成/未完成项及可闭环架构计划。
- `PRODUCT_GOALS_EXECUTION_PLAN.md`：产品目标实施计划，明确总提纲目标、阶段状态与闭环执行模板。

## 使用建议
- 新增需求前，先对齐 `BUSINESS_DIRECTION.md`。
- 设计技术方案时，对照 `TECHNICAL_ROADMAP.md` 与 `ARCHITECTURE_MAINTENANCE.md`。
- 每完成一个迭代，更新 `PRODUCT_GOALS_EXECUTION_PLAN.md` 的“已完成/未完成/下一步闭环计划”。
